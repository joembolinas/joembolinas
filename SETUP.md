# Setup — joembolinas GitHub Profile README

## 1. Repo structure
Create a public repo named **exactly your GitHub username** (e.g. `joembolinas/joembolinas`)
— that's what makes GitHub render it as your profile README. Commit these into it:

```
your-username/
├── README.md
└── assets/
    └── svg/        ← all 31 generated .svg files go here
```

The `generator/` folder (generate.py + svgkit.py) is just the tool that built the SVGs —
you don't need to commit it, but keep it if you want to tweak text/colors and re-run it later.

## 2. Find-and-replace before publishing
Search `README.md` for these placeholders and swap in your real info:

| Placeholder | Replace with |
|---|---|
| `YOUR_GITHUB_USERNAME` | your real GitHub username (appears ~5 times — view counter, activity graph, snake, stats card, streak card) |
| `joembolinas23@gmail.com` | your real email (also update the `mailto:` href) |
| `linkedin.com/in/joembolinas` | your real LinkedIn |
| `github.com/joembolinas` | your real GitHub URL |
| `REPO_ONE` … `REPO_SIX` | your actual repo links for the 6 featured projects |

## 3. The contribution snake (optional, needs a one-time GitHub Action)
`watch -n 1 progress` → "Contribution Snake" is generated by a GitHub Action, not a static
URL. If you want it working, add Platane's `snk` action to a workflow in your profile repo
(search "github contribution snake action" for the 10-line YAML) — otherwise just delete
that `<picture>` block from the README.

## 4. Editing content (numbers, timeline, bio, project cards, etc.)
Everything text-based lives in `generator/generate.py` as plain Python data (e.g. the
`TIMELINE`, `PROJECTS`, `LANGS` lists, the mission-log lines). Edit the values there, then:

```bash
cd generator
python3 generate.py
```

This regenerates every SVG in `assets/svg/` in place — no design tool needed.

## 5. Why this approach
- The 9 section headers (`> whoami`, `> ls --color=auto ./skills`, …) are **custom typing-animated
  SVGs** (no third-party typing-svg service dependency) — they type out, pause, erase, and loop.
- Bio, mission log, career timeline, finger output, project cards, skill groups, stat cards,
  language chart, and contact bar are **custom terminal-styled SVG panels**, hosted in your
  own repo (`assets/svg/`) so they never break if an external badge service goes down.
- GitHub Activity Graph, the contribution Snake, full stats card, and streak card stay as
  **live external widgets** (vercel/herokuapp/komarev) since those need to query the GitHub
  API in real time — that can't be baked into a static SVG.
